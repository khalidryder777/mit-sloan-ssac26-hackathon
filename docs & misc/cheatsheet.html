Feature Selection Strategy

Include

  • Behavioral: attendance, spend, streaming
  • Commercial: brand affinity, ad receptivity
  • Psychographic: values, intensity, social

Exclude

  • IDs (respondent_id, etc.)
  • Demographics (age, gender, region)
  • Near-zero-variance columns
  • Highly correlated pairs (r > 0.95)

Why Exclude Demographics?

Including age/gender gives you boring demographic clusters. Excluding them gives you behavioral fan types. If demographics correlate after clustering, that validates the finding without tautology.

Algorithm Decision Tree
Data has categorical columns?

Algorithm Comparison

K-Means The Workhorse

  • Use when: Numeric data, roughly spherical clusters
  • Pros: Fast, deterministic, easy to explain
  • Cons: Must pick k, assumes spherical, outlier-sensitive
  • Key param: n_init=10, random_state=42

K-Prototypes The Hybrid

  • Use when: Mixed numeric + categorical data
  • Pros: Handles both types natively
  • Cons: Slower, must specify categorical_indices
  • Key param: init='Huang', n_init=5

HDBSCAN The Rebel

  • Use when: Unknown k, non-spherical clusters, want outlier detection
  • Pros: Auto k, finds weird shapes, flags noise
  • Cons: Sometimes everything is noise, hard to tune
  • Key param: min_cluster_size=n//50

Percentile The Emergency Chute

  • Use when: Nothing else works, running out of time
  • How: Top 2-3 features → tertiles → cross-tab → merge
  • Pros: Always works, always interpretable
  • Cons: Not sophisticated, but beats nothing
Validation: Are Your Clusters Real?

Silhouette Score

> 0.3

Range: [-1, +1]. Higher = more distinct.
> 0.5 good • > 0.7 excellent • < 0.25 suspicious

Stability (ARI)

> 0.7

10 runs, different seeds.
> 0.9 rock solid • > 0.7 good enough • < 0.7 sketchy

Interpretability

Can you name it?

If you can describe each cluster in one sentence a non-technical person gets → it's real.

Commercial Value Index
CVI = 0.30(Spending) + 0.25(Brand Receptivity) + 0.20(Engagement) + 0.15(Social Amplification) + 0.10(Growth Potential)

Each sub-score = average of mapped columns, min-max normalized to [0, 1]

Why These Weights?

0.30 Spending Score

Direct monetizable signal. Ticket spend, merch spend. Money talks — current spending is the best predictor of future spending.

0.25 Brand Receptivity

Sponsor awareness, purchase intent, brand affinity. This is what sponsors are literally buying. Can they reach this fan?

0.20 Engagement Depth

Games attended, streaming hours, app usage, fandom intensity. Deep engagement = retention = lifetime value.

0.15 Social Amplification

Shares, posts, recommendations. A fan who amplifies is a walking billboard — worth more than their own spending.

0.10 Growth Potential

Youngest fans, newest fans, highest ad receptivity. Low weight because it's speculative, but it represents future value. Consider inverting age and tenure.

When Things Go Wrong (And They Will)
"The data makes no sense"
  1. Run df.describe(include='all') — ranges reveal meaning (1-5 = Likert, 0/1 = binary)
  2. Check df.nunique() — 2 uniques = binary, 5-7 = Likert, 1 = useless
  3. Ask the organizers: "What's column Q14?" is a legitimate question
  4. Nuclear: treat everything numeric, standardize, K-Means it. You'll find patterns.
"Clustering gives random garbage"
  1. Reduce features. PCA to 5-10 components, then cluster.
  2. Try different algo. K-Means failing? Hierarchical. Everything? HDBSCAN.
  3. Check outliers. A few extreme values can eat your centroids.
  4. Nuclear: percentile segmentation on top 3 highest-variance columns.
"My code crashed mid-pipeline"
  1. Notebooks are re-runnable from any cell. Data saves at each stage.
  2. Common crashes: NaN survived cleaning → df.isnull().sum()
  3. Mixed types in a column → df[col].apply(type).value_counts()
  4. Memory error → df.sample(10000) to reduce dataset
"Streamlit won't launch"
  1. streamlit run app/dashboard.py from PROJECT ROOT
  2. Port busy? --server.port 8502
  3. Crashes on load? Check data/clustered_data.csv exists.
  4. Nuclear: skip Streamlit. Charts are PNGs in outputs/. Use those in slides.
"UMAP won't install / crashes"

Use PCA instead: reduce_dimensions(X, method='pca'). Less pretty, equally functional. The pipeline handles this fallback automatically.

Q&A Cheat Sheet
"Why K-Means and not another algorithm?"

"Best silhouette score, most interpretable clusters. Stability check: 10 runs, ARI of 0.87. These clusters are robust, not artifacts."

If they push: "HDBSCAN shines on spatial data with noise; this is clean survey data with ordinal scales. K-Means is the right tool."

"How did you choose k?"

"Silhouette peaked at k=5, Calinski-Harabasz confirmed, elbow showed diminishing returns. But the strongest argument is interpretability — at k=4 we lost a distinction, at k=6 one cluster was too small."

"Why those CVI weights?"

"Spending highest at 0.30 because it's the most direct measure. We ran sensitivity — as long as spending > 0.20 and social < 0.25, rank order doesn't change. Finding is robust."

"Aren't you just finding demographic clusters?"

"No — by design. We excluded age, gender, and region from clustering features. Clusters are defined by behavior and psychographics. Demographics correlate AFTER — that's validation, not tautology."

"What would you do with more time?"

"Three things: validate segments with a follow-up survey, A/B test Silent Superfan outreach (CRM vs. social), track segment migration over time to understand conversion triggers."

Don't know the answer?

"That's a great question and exactly the direction our next analysis would take." Then move on. Never make up a number.

Quick-Reference Formulas

Silhouette Score

s(i) = (b(i) - a(i)) / max(a(i), b(i))

a(i) = mean distance to same cluster
b(i) = mean distance to nearest other cluster
Range: [-1, +1], higher = better

Calinski-Harabasz

CH = [SS_B / (k-1)] / [SS_W / (n-k)]

SS_B = between-cluster sum of squares
SS_W = within-cluster sum of squares
Higher = better (compare across k values)

Commercial Value Index

CVI = 0.30(S) + 0.25(B) + 0.20(E) + 0.15(A) + 0.10(G)

S = Spending Score       (ticket + merch)
B = Brand Receptivity    (awareness + intent)
E = Engagement Depth     (attendance + streaming)
A = Social Amplification (shares + recommends)
G = Growth Potential     (youth + newness)

Each sub-score: min-max normalized to [0, 1]

Index Value

Index = (cluster_mean / population_mean) x 100

  > 120 = over-indexes (stands out)
  < 80  = under-indexes (below average)
  80-120 = meh, average

Adjusted Rand Index (Stability)

ARI in [-1, 1]

1   = perfect agreement between runs
0   = random (no better than chance)
< 0 = worse than random (yikes)

Usage: 10 runs, compare all pairs,
       take mean ARI. Want > 0.7.

Value Efficiency

Efficiency = pct_of_value / pct_of_size

  > 1.0 = punching above weight
  = 1.0 = fair share
  < 1.0 = underperforming

Example: 19% of fans, 25% of value
  => 25/19 = 1.32x efficiency
